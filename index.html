How Brique Finder works

🔍 Search
Type any item (e.g. iphone 12, ps5) and click Search. The server opens Facebook Marketplace in Curitiba, captures all listings, and shows them ranked by score. Each search is saved to history.
📊 Score (0–100)
Every listing gets a score made of two parts:
Discount (0–60 pts) — how much cheaper it is vs. the average price for that keyword. 40% below average = 60 pts.
Preference (−20 to +40 pts) — how similar the price is to items you liked (👍) vs. disliked (👎). The more you vote, the smarter it gets.
💰 Profit estimate
Shows avg price − listing price as estimated flip profit and margin %. Green = ≥40% margin, yellow = ≥20%. The average gets more accurate each time you search for the same keyword.
👍 👎 Like / Dislike
Vote on listings to train the score algorithm. Liked items teach it the price range you're happy to buy at. Disliked items make it penalise similar prices in the future. Votes are shared across all users on the same server.
📡 Live Feed
Add keywords to monitor (e.g. macbook, xbox). Click Start Auto-Scan and the app will re-scan all keywords every 5 minutes automatically, showing the best-scored offers at the top.
🕓 History
Every search is saved. The dropdown shows the keyword, how many listings were found, who searched (by their key label), and the exact date/time. Load any past search to see what was available then.
🔑 Keys
Access is controlled by keys. The 🔑 button (admin only) lets you create a key for each friend with their name as label, and revoke it at any time. History will show their name next to every search they made.
⚙️ Server URL
The ⚙️ button sets which server the frontend talks to. Default is http://localhost:5001. If sharing via ngrok, friends paste the ngrok URL here once and it's saved in their browser.
